## Итоговая таблица: JavaScript vs C#

| Аспект            | C#                                          | JavaScript                                      |
| ----------------- | ------------------------------------------- | ----------------------------------------------- |
| **Типизация**     | Статическая                                 | Динамическая                                    |
| **Компиляция**    | Да (в IL-код)                               | Нет (интерпретация / JIT)                       |
| **Точка входа**   | `Main()`                                    | Отсутствует (код выполняется при загрузке)      |
| **Переменные**    | `int x = 5;`                                | `let x = 5;`                                    |
| **Константы**     | `const int X = 5;`                          | `const X = 5;` (для ссылочных типов — хитрости) |
| **Вывод в консоль** | `Console.WriteLine()`                       | `console.log()`                                 |
| **Проверка типа** | Во время компиляции                         | `typeof`                                         |
| **Сравнение**     | `==` (по значению)                          | `==` (нестрогое) и `===` (строгое)               |
| **Условия**       | `if`/`else`/`switch`                        | `if`/`else`/`switch` + тернарный оператор       |
| **Циклы**         | `for`/`while`/`do-while`/`foreach`          | `for`/`while`/`do-while`/`for...of`             |
| **Функции**       | Методы с указанием типов                    | Функции без типов (`function`, стрелочные)      |
| **Массивы**       | `int[] arr = new int[5];`                   | `let arr = [];`                                  |
| **Объекты**       | Требуют класс                               | Можно создать литералом `{}`                     |
| **ООП**           | Классическое (классы, наследование)         | Прототипное (классы — синтаксический сахар)     |
| **Null**          | `null`                                      | `null` и `undefined`                            |
| **Строгость**     | Высокая (компилятор не пропустит ошибки)   | Низкая (требуется дисциплина разработчика)      |

## Главные выводы

1.  **JavaScript гибче C#** — ключевое отличие заключается в динамической типизации. Переменная может менять свой тип в рантайме, что дает большую свободу, но требует аккуратности.
2.  **JavaScript проще синтаксически** — для объявления переменной не нужно указывать тип, а для создания объекта достаточно фигурных скобок. Это снижает порог входа.
3.  **JavaScript опаснее** — из-за отсутствия компиляции многие ошибки (например, несоответствие типов) всплывают только во время выполнения программы, что усложняет отладку.
4.  **C# безопаснее** — строгая типизация и компилятор на стороне разработчика отсекают огромный пласт потенциальных ошибок еще на этапе написания кода, делая его более надежным и предсказуемым.
